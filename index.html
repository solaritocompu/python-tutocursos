Como navegar por el sitio?

En computadora haz click con el mouse en los enlaces de la barra de navegación arriba para leer la sección que elijas y para volver arriba presiona la tecla home. En celular y tableta arrastra el dedo.

Requisitos para el curso

  1. Tener una computadora y saber usarla también puede ser un celular o una tableta.
  2. Saber usar un editor de texto. No te quejes! Bendito... Quieres programar y no sabes como guardar un archivo de texto. Mejor busca en Internet como hacerlo o coge un curso de computadora antes de seguir este curso. Hay que gatear antes que caminar y caminar antes que correr.
  3. Instalar el Thonny porque incluye el editor y python en un solo instalador. Viene listo para escribir archivos de código sin tener que configurarlo y tiene un área para ejecutar instrucciones en el shell. Web de Thonny para descargarlo e instalarlo

Mostrar texto en el shell

Si abres Thonny se vera lo que se llama el shell abajo:


Python 3.7.9 (bundled)
>>>

Escribe print("Hola mundo") después de >>> y presiona enter. Veras que aparece en el shell Hola mundo o cualquier carácter o numero que introduzcas entre las comillas. La instrucción print significa imprime así que le estamos diciendo a la computadora imprime Hola mundo y ella lo hace. Esta importante instrucción es la primera que se aprende, sirve para mostrar texto y también sirve para mostrar el contenido de variables lo que es muy útil y lo explicare mas adelante en el curso.

Ejemplo: 

>>> print("Hola mundo")
Hola mundo
>>>

Escribir comentarios en el código.

Los comentarios son una forma de explicar como funciona una sección de código, por si necesitas editar tu programa el próximo mes te ayude a recordar como funciona una o varias instrucciones. Los comentarios se escriben así:


#Esto es un comentario y es solo para que alguien lo lea.

Si escribes algo que no sea una instrucción valida sin # delante y ejecutas, nos dará un error y el programa se detendrá.

Para que sirven los comentarios?

Los comentarios sirven para dejar sin efecto una o varias lineas de código y así el interprete python no las ejecute. Si estas indeciso entre una linea u otra puedes comentar una y probar la otra.


#Cuando se ejecute el programa las lineas con # delante no serán 
#ejecutadas por el interprete Python y las tratara como si no 
#estuvieran escritas.
print("Se ejecutara solamente esta linea cuando corra el programa")
#print("Esta linea sera tratada como si no estuviera escrita")

Si quieres invalidar muchas lineas de código para que el interprete no las ejecute o escribir un párrafo de comentario se escribe un comentario múltiple así:


''' Esto es un comentario multilínea,
sirve para documentar una parte
del programa, y se usa comúnmente
para documentar una función.'''

Puedes ponerlo entre tres comillas simples ''' o dobles """.

Ejemplos de uso de comentarios para el área de texto


import math #importamos el modulo math
print(math.pow(5, 2)) #Muestra 5 al cuadrado

    '''Esta funcion devuelve la suma de num1 y num2'''
    return num1 + num2

Declarar variables

Una variable sirve para ponerle un nombre a un dato.


>>> x = 5

Guardamos el dato 5 en la variable x con el operador de asignación que es un signo de igual y después se podrán realizar operaciones matemáticas como x + 7 con ella. Con esta instrucción la variable x esta definida, declarada e inicializada con el valor 5.

Práctica de operaciones matemáticas con variables


>>> x = 5
>>> print(x + 7) #suma
12
>>> print(x - 2) #resta
3
>>> print(4 * x) #multiplicación
20
>>> print(9 / x) #división
1.8
>>> print(9 // x) #división entera
1
>>> print(x ** 2) #exponente
25
>>> x = x + 1 #Aumentamos uno el valor de x
>>> print(x)
6
>>> x += 1 #Es lo mismo que x = x + 1 abreviado, hace lo mismo.
>>> print(x)
7

La segunda linea da 12 porque x es 5 y 5 + 7 es 12.

Tipos de variable y de datos

Si una variable contiene un dato de tipo float es de tipo float.


a = 5 #integer
b = 7.35 #float
c = "texto escrito" #string
d = [1, 2, 3, 4, 5] #list
e = (7,6 5) #tuple
f = {1:"uno", 2:"dos", 3:"test"} #dictionary
g = True #boolean

Integer son números enteros como 1 o 56 o 735.

Float son números con parte decimal, flotante como 356.7 o 10.0 o 4.5.

String son cadenas de caracteres como "Roca" o "La puerta del garaje" o "asdf%^(1234". Los string o cadenas pueden ser numéricos, alfanuméricos y caracteres especiales. En Python pueden estar entre comillas dobles o simples mas info. en la sección de strings.

Las listas son los llamados vectores, arreglos o matrices en otros lenguajes permiten guardar secuencias de datos para iterar sobre ellos. Mas información sobre las listas en la sección de listas y mas sobre la iteración en la sección de bucles.

Las tuplas son como las listas pero no se puede modificar su valor una vez definidas y tienen menos métodos que las listas pero se usan siempre que se puede en su lugar de porque son mas optimas que las listas en el uso de los recursos de la computadora.

Los diccionarios son llamados HashMap en otros lenguajes. Sirven para guardar valores con una key o clave.

Los boolean o booleanos son valores que pueden ser True o False , verdaderos o falsos, muy usados en condicionales.

Expresiones booleanas

Una expresión booleana es una expresión que se evalúa y devuelve como resultado un valor booleano, que es un valor que es o True o False, o Verdadero o Falso. Nótese que en Python los valores booleanos empiezan con mayúscula. Las expresiones booleanas se usan en condicionales, y se crean mediante el uso de operadores de comparación, conocidos como operadores relacionales, que son:

== igual que
=! no es igual que
> mayor que
< menor que
>= mayor o igual que
<= menor o igual que

Operador igual que

Con un operador de igual que, si lo que esta a ambos lados del operador es igual devuelve True, en caso contrario devuelve False. Fíjese que el operador de igualdad se escribe con dos signos de igual, y el operador de asignación con uno solo.


>>> h = 10
>>> h == h
True
>>> h == 3
False

En la primera linea asignamos 10 a h por eso h es 10. En la segunda linea se evalúa si h es igual a h lo que devuelve True en la tercera porque es verdadero que 10 es igual a 10. En la cuarta linea se evalúa si h es igual a 3 lo que devuelve False porque 10 no es igual a 3.

Operador no es igual que

Con un operador de no es igual, si lo que esta a ambos lados del operador es igual devuelve False, en caso contrario devuelve True.


>>> h = 10
>>> h != h
False
>>> h != 3
True

En la segunda linea se evalúa si h no es igual a h lo que devuelve False en la tercera porque es falso que 10 no es igual a 10. En la cuarta linea se evalúa si h no es igual a 3 lo que devuelve True porque 10 no es igual a 3.

Operador mayor que

Con un operador de mayor que si lo que esta a la izquierda del operador es mayor que lo que esta a la derecha devuelve True, en caso contrario devuelve False, y si lo que esta a la izquierda es igual que lo que esta a la derecha devuelve False.


>>> num1 = 2
>>> num2 = 9
>>> num1 > num2
False
>>> num1 > 1
True

En la tercera linea evalúa si num1 es mayor que num2 lo que devuelve False porque 2 no es mayor que 9. En la quinta linea evalúa si num1 es mayor que 1 lo que devuelve True porque 2 es mayor que 1.

Operador menor que

Con un operador de menor que si lo que esta a la izquierda del operador es menor que lo que esta a la derecha devuelve True, en caso contrario devuelve False, y si lo que esta a la izquierda es igual que lo que esta a la derecha devuelve False.


>>> num1 = 2
>>> num2 = 9
>>> num1 < num2
True
>>> num1 < 1
False

En la tercera linea de evalúa si num1 es menor que num2 lo que devuelve True porque 2 es menor que 9. En la quinta linea evalúa si num1 es menor que 1 lo que devuelve False porque 2 no es menor que 1.

Operador mayor o igual que

Con un operador de mayor o igual que si lo que esta a la izquierda del operador es mayor o igual que lo que esta a la derecha devuelve True, y solo si lo que esta a la izquierda del operador es menor que lo que esta a la derecha devuelve False.


>>> 600 >= 500
True
>>> 500 >= 500
True
>>> 400 >= 500
False

En la primera linea se evalúa si 600 es mayor o igual que 500 lo que devuelve True porque 600 es mayor que 500. En la tercera linea se evalúa si 500 es mayor o igual que 500 lo que devuelve True porque 500 es igual que 500. En la quinta linea se evalúa si 400 es mayor o igual que 500 lo que devuelve False porque 400 no es mayor ni igual que 500.

Operador menor o igual que

Con un operador de menor o igual que si lo que esta a la izquierda del operador es menor o igual que lo que esta a la derecha devuelve True, y solo si lo que esta a la izquierda del operador es mayor que lo que esta a la derecha devuelve False.


>>> 600 <= 500
False
>>> 500 <= 500
True
>>> 400 <= 500
True

En la primera linea se evalúa si 600 es menor o igual que 500 lo que devuelve False porque 600 no es menor ni igual que 500. En la tercera linea se evalúa si 500 es menor o igual que 500 lo que devuelve True porque 500 es igual que 500. En la quinta linea se evalúa si 400 es menor o igual que 500 lo que devuelve True porque 400 es menor 500.

Operadores lógicos

Operadores que nos permite modificar expresiones booleanas uniéndolas o negándolas para que al ser evaluadas den un como resultado un valor booleano se conocen como operadores lógicos, y son:

and
or
not

Operador and

Con un operador de and si lo que esta a ambos lados del operador es True devuelve True, si lo que esta en un lado o ambos lados es False devuelve False


>>> True and 1 == 1
True
>>> 5 != 5 and 4 >= 4
False
>>> "a" != "a" and False
False

En la primera linea la evaluación de True devuelve True al lado izquierdo del operador y la evaluación de 1 == 1 devuelve true al lado derecho del operador por lo tanto como lo que esta a ambos lados del operador and es True la evaluación del and devuelve True en la segunda linea. En la tercera linea la evaluación de 5 != 5 devuelve False al lado izquierdo y la evaluación de 4 >= 4 devuelve True en el lado derecho por lo tanto como lo que esta en un lado del operador and es False la evaluación del and devuelve False en la cuarta linea. En la quinta linea como lo que esta a ambos lados del and es falso devuelve False.

Operador or

Con el operador or si lo que esta a un lado del operador o a ambos lados es True devuelve True, solamente devuelve False si lo que esta a ambos lados del operador es False.


>>> False or True
True
>>> 7 > 4 or 1 < 9
True
>>> 45 > 200 or 34 != 34
False

En la primera linea la evaluación de or devuelve True porque lo que esta a un lado del or es True. En la tercera linea también devuelve True porque lo que esta a ambos lados del or es True. En la quinta linea devuelve False porque lo que esta a ambos lados del or es False.

Operador not

Con un operador not se hace que la evaluación que devuelve True devuelva False y la que devuelve False devuelva True negando la expresión. Otra forma mas simple de decirlo es not convierte True en False y False en True.


>>> not True
False
>>> not False
True

En el ejemplo anterior not convirtió True en False en la primera linea y False en True en la tercera.

La sentencia if

Los condicionales se crean con la sentencia if, que se usa para determinar el flujo de ejecución del programa, basándose en el resultado de la evaluación de una expresión booleana, que puede dar como resultado True o False, Verdadero o Falso. Si la expresión booleana es verdadera, se ejecutan las instrucciones indentadas después del if, si la condición es falsa saltara las instrucciones indentadas y seguirá ejecutando las que siguen.


gravedad = 0
if gravedad < 0:
    print("Cuidado que te elevas!")

La indentación o sangrado determina si las lineas de código están dentro de un bloque.


velocidad = 5000
if velocidad == 5000:
    print("Este print y los próximos 2 se ejecutan si la ", end='') 
    print("condición es verdadera.",)
    print("Vas muy rápido")
print("Como estas instrucciones están fuera del if ", end = '')
print("se ejecutan si la condición es verdadera y si es falsa.")

Los 3 print indentados después del if se conocen como un bloque. Si quieres probar las instrucciones del if cuando estas haciendo un programa ponle True en el área de la condición y se ejecutaran siempre que corras el programa.


if True:
    print("Esta linea siempre se ejecutara.")

if False:
    print("Esta linea nunca se ejecutara.")

Si quieres saltarte el if y que nunca se ejecuten sus instrucciones mientras estas haciendo pruebas ponle False. 


if 1:
    print("Esta linea siempre se ejecutara.")

if 0:
    print("Esta linea nunca se ejecutara.")

También se puede poner 1 en lugar de True y 0 en lugar de False.

La sentencia else

Si la condición del if, su expresión booleana es falsa se ejecutaran las instrucciones indentadas después del else.


edad = 16
if edad > 18:
    print("Es mayor de edad")
else:
    print("Es menor de edad")

Si ejecutas el programa imprime es menor de edad porque la condición del if es falsa

La sentencia elif

Si la condición del if es True se ejecuta el bloque del if y se salta el elif y el else pero si la condición del if es False se evalúa la condición del elif y si resulta ser verdadera se ejecuta el bloque del elif saltándose el bloque del else y si la condición del elif también es falsa ejecuta las instrucciones del bloque del else. Ejemplificado abajo para que se entienda.


altura = 6
if altura >= 6:
    print("alto")
elif altura == 5:
    print("mediano")
else:
    print("chiquito")

Como la altura es 6 la condición del if es True y se muestra alto. Fíjese como se salto las instrucciones del elif y del else al no ejecutarlas.


altura = 5
if altura >= 6:
    print("alto")
elif altura == 5:
    print("mediano")
else:
    print("chiquito")

Aquí la altura es 5 la expresión del if devuelve False por lo tanto su bloque no se ejecuta y evalúa la expresión del elif que resulta ser True por lo que se ejecuta su bloque mostrando mediano y se salta el bloque del else. Fíjese como se salto las instrucciones del if y del else al no ejecutarlas.


altura = 4
if altura >= 6:
    print("alto")
elif altura == 5:
    print("mediano")
else:
    print("chiquito")

Con una altura de 4 la condición del if es False por lo que no se ejecuta su bloque la del elif también es False por lo que tampoco su bloque se ejecuta y en como todas las condiciones anteriores son False se ejecuta el bloque del else.
